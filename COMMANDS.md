# Руководство по игровым командам mod-hardcore

## Обзор

Модуль предоставляет набор команд для проверки статуса хардкор-режима и получения информации о его особенностях.

---

## Список команд

### `.hardcore status`

**Описание:** Проверяет текущий статус хардкор-режима вашего персонажа.

**Уровень доступа:** Игрок (SEC_PLAYER)

**Синтаксис:**
```
.hardcore status
```

**Выводит:**
- Активность режима (АКТИВЕН/НЕ АКТИВЕН)
- Текущий уровень персонажа
- Состояние персонажа (ЖИВ/ПОГИБ НАВСЕГДА)
- Статус окончательной смерти (активна/отключена)
- Уровень отключения режима (если настроено)
- Оставшееся время кулдауна подземелий

**Пример вывода для активного режима:**
```
╔══════════════════════════════════════╗
║     СТАТУС ХАРДКОР-РЕЖИМА           ║
╚══════════════════════════════════════╝
Режим: АКТИВЕН
Уровень: 45
Статус: ЖИВ
Окончательная смерть до уровня: 60
Режим завершится на уровне: 80
Кулдаун подземелий: 12 ч. 35 мин.
```

**Пример вывода для неактивного режима:**
```
╔══════════════════════════════════════╗
║     СТАТУС ХАРДКОР-РЕЖИМА           ║
╚══════════════════════════════════════╝
Режим: НЕ АКТИВЕН
Используйте игровой объект для активации.
```

---

### `.hardcore info`

**Описание:** Показывает подробную информацию о хардкор-режиме и его настройках на сервере.

**Уровень доступа:** Игрок (SEC_PLAYER)

**Синтаксис:**
```
.hardcore info
```

**Выводит:**
- Общее описание режима
- Модификаторы урона и опыта
- Список всех ограничений
- Информацию об уровнях отключения
- Правила групповой игры

**Пример вывода:**
```
╔══════════════════════════════════════╗
║   ИНФОРМАЦИЯ О ХАРДКОР-РЕЖИМЕ       ║
╚══════════════════════════════════════╝
Режим «Без права на ошибку»

Особенности:
• Только одна жизнь
• Смерть необратима
• Воскрешение невозможно

Модификаторы:
• Урон: 90%
• Получаемый урон: 150%
• Опыт: 100%

Ограничения:
• Рыцари смерти запрещены
• Аукцион заблокирован
• Почта заблокирована
• Гильдейский банк заблокирован
• Поля боя заблокированы
• Арена заблокирована
• Принудительный PvE
• Группы: ±5 уровней
• Кулдаун подземелий: 24 ч.

Окончательная смерть до 60 уровня
Автоотключение на 80 уровне

Используйте .hardcore status для проверки вашего статуса.
```

## Примеры использования

### Проверка перед входом в подземелье

**Ситуация:** Вы хотите узнать, можете ли войти в подземелье.

**Команда:**
```
.hardcore status
```

**Результат:**
- Если кулдаун активен: увидите оставшееся время
- Если кулдаун истёк: увидите "готово"

---

### Узнать правила сервера

**Ситуация:** Вы новый игрок и хотите узнать о режиме хардкор.

**Команда:**
```
.hardcore info
```

**Результат:**
- Полная информация о модификаторах
- Список всех ограничений
- Правила окончательной смерти

---

### Проверка после смерти

**Ситуация:** Вы умерли и хотите узнать, можете ли воскреснуть.

**Команда:**
```
.hardcore status
```

**Результат:**
- Если уровень ниже MaxDeathLevel: "ПОГИБ НАВСЕГДА"
- Если уровень выше: информация о возможности воскрешения

## Часто задаваемые вопросы

### Можно ли использовать команды на любом уровне?

Да, команды доступны всем игрокам независимо от уровня и статуса хардкор-режима.

### Что делать, если команда не работает?

1. Убедитесь, что модуль включен (`Hardcore.Enable = 1` в конфиге)
2. Проверьте, что сервер пересобран с новой версией модуля
3. Проверьте логи сервера на наличие ошибок
4. Убедитесь, что используете правильный синтаксис (с точкой в начале)

### Есть ли команды для администраторов?

В текущей версии (1.1.0) специальные команды для администраторов не реализованы. Планируется добавить:
- `.hardcore enable <player>` - Включить режим для игрока
- `.hardcore disable <player>` - Отключить режим для игрока
- `.hardcore reset <player>` - Сбросить кулдауны
- `.hardcore stats` - Статистика по всем хардкор-игрокам

### Можно ли добавить свои команды?

Да! Модуль использует стандартную систему команд AzerothCore. Вы можете добавить свои команды в `HardcoreCommandScript.cpp` и пересобрать модуль.

## Технические детали

### Безопасность

- Команды используют стандартную систему безопасности AC
- Уровень доступа: `SEC_PLAYER` (доступны всем игрокам)
- Команды не консольные (`Console::No`)

### Производительность

- Команды выполняются мгновенно
- Запросы к базе данных минимальны (только для чтения настроек)
- Не создают нагрузки на сервер

### Совместимость

- Работают на всех поддерживаемых платформах (Windows/Linux)
- Совместимы с другими модулями
- Не конфликтуют со стандартными командами AC
